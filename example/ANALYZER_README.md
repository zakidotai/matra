# Material Science Table Analyzer

This toolkit analyzes research paper tables to extract material compositions and spall strength data using a hosted LLM (Llama-2-chat).

## Features

- **Table Analysis**: Identifies material compositions and spall strength values in research paper tables
- **Cross-Table Matching**: Matches materials between different tables in the same paper
- **JSON Output**: Structured output with confidence scores and metadata
- **Streamlit Dashboard**: Web interface for interactive analysis
- **Batch Processing**: Analyze multiple papers automatically

## Files

- `material_analyzer.py` - Core analysis engine
- `streamlit_dashboard.py` - Web dashboard interface
- `test_analyzer.py` - Test script for validation
- `requirements.txt` - Python dependencies

## Setup

1. **Install Dependencies**:
```bash
pip install -r requirements.txt
```

2. **Start LLM Server**:
Ensure your Llama-2-chat model is running on `http://localhost:8000`

3. **Prepare Data**:
Make sure `corpus_spall/pii_table_dict.pkl` exists (generated by `get_tables_from_xmls.py`)

## Usage

### Command Line Interface

```bash
# Basic analysis
python material_analyzer.py --pkl_path corpus_spall/pii_table_dict.pkl

# Custom output file
python material_analyzer.py --pkl_path corpus_spall/pii_table_dict.pkl --output results.json

# Custom LLM server
python material_analyzer.py --pkl_path corpus_spall/pii_table_dict.pkl --llm_url http://your-server:8000
```

### Streamlit Dashboard

```bash
# Launch web interface
streamlit run streamlit_dashboard.py
```

The dashboard provides:
- File upload interface
- Real-time analysis progress
- Interactive visualizations
- Detailed results exploration
- Data export capabilities

### Testing

```bash
# Test LLM connection and analyzer
python test_analyzer.py
```

## Analysis Process

1. **Table Analysis**: Each table is analyzed individually to identify:
   - Material compositions (element percentages, alloy compositions)
   - Spall strength values (dynamic strength, impact resistance)
   - Sample identifiers for cross-table matching

2. **Cross-Table Matching**: Materials are matched between tables based on:
   - Common identifiers (sample names, IDs)
   - Composition similarity
   - Property correlations

3. **Output Generation**: Results are structured into:
   - Matched materials (composition + property)
   - Unmatched compositions (composition only)
   - Unmatched properties (property only)

## Configuration

### LLM Server Settings

The analyzer expects a vLLM server with the following configuration:
- Model: Llama-2-chat
- Endpoint: `/v1/chat/completions`
- Temperature: 0.1 (for consistent JSON output)
- Max tokens: 2000

### Analysis Parameters

- **Max Papers**: Limit analysis for testing (default: all papers)
- **Confidence Threshold**: Minimum confidence for matches (default: 0.0)
- **Timeout**: LLM request timeout (default: 30 seconds)

## Troubleshooting

### Common Issues

1. **LLM Connection Failed**:
   - Check server is running on correct port
   - Verify model is loaded
   - Test with `test_analyzer.py`

2. **Invalid JSON Response**:
   - LLM may not be following prompt format
   - Try adjusting temperature or prompt
   - Check server logs

3. **Empty Results**:
   - Verify pickle file contains valid data
   - Check table extraction was successful
   - Review LLM responses for errors

### Performance Tips

- Use `max_papers` parameter for testing
- Process papers in batches for large datasets
- Monitor LLM server resources
- Cache results for repeated analysis
